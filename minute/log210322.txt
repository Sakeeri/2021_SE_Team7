21.03.22 회의록

개발 주제, 개발 도구 결정 및 온라인 쇼핑몰의 아키텍쳐/개발 프로세스에 대한 회의



1. 어떤 언어를 사용할 것인가?

   웹 사이트 개발의 기본적인 언어인 HTML, CSS, Javascript와 서버 간 통신을 위한 PHP, DB 관리를 위한 Mysql을 사용할 예정이다.
   LAMP 스택을 사용 중이기에 PHP를 채택했으며, Mysql은 MariaDB로 변경될 수도 있다.



2. 어떤 라이브러리/프레임워크를 사용할 것인가?

   여러 후보들 중 Bootstrap, React, Node.js, jQuery 등으로 좁혀졌다.
   다음 회의 전까지 위 도구들을 사용해 제작한 웹 사이트의 예시 등을 찾아보고, 해당 도구를 사용해 어떻게 개발을 진행할 것인지에 대해 대략적인 구상을 하기로 했다.



3. 어떤 쇼핑몰을 만들 것인가?

   굿즈 쇼핑몰, 서점, 식자재 쇼핑몰 등의 의견 중 식자재 쇼핑몰로 결정되었다. 하지만 추후 공개될 요구사항에 따라 주제를 변경할 수도 있다.



4. 프론트/백엔드 업무 구분은 1~2주 정도 탐색기를 거친 후 결정

   업무를 구분한다면 각자의 영역만 신경쓰면 되기에 편리하지만, 팀원 간 소통의 중요성이 높아지고 각 기능의 마감 기한을 조율해야 한다. 
   업무를 구분하지 않는다면 팀원 각자가 웹 페이지에 대한 전반적인 이해도가 높아지겠지만, 넓은 영역을 모두 신경써야 하고 새로 배울 내용이 많아진다.
   1~2주의 유예기간을 두고, 2번 과정을 거치면서 프론트/백엔드에 대한 이해와 업무 분장에 대한 각자의 의견을 정리하기로 하였다.
   개발 영역에 대한 결론이 난 후에야 sprint cycle을 정할 수 있을 것 같다.



5. 예상되는 요구사항

   1주차 강의자료에서 예시로 제시되었던 요구사항을 토대로, 프로젝트에 제시될 요구사항들을 추측하였다.
   예시로 제시된 요구사항은 메인 화면/관리자 화면/상품리스트 화면/결제 화면이다. 추가로 물품 수량관리 화면과 회원가입/로그인 화면, 장바구니 화면을 요구사항으로 예상하였다.
   예상된 요구사항들에 대해 우선순위를 매기고 개발 절차를 진행한다.
   기능 명세 결정 -> 기능 제작 -> 완성 확인/테스트 -> 다음 기능 제작 -> 완성 확인/테스트 -> 이전 기능에 병합 후 테스트 -> 다음 기능 제작
   위 과정의 반복을 통해 점진적으로 기능을 추가해 완성해 나가기로 하였다.
